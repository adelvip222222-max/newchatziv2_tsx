# Chatzi CRM Ticket Flow, Bot Settings, Categories, and Billing Usage Update

## Core CRM behavior

- Added a CRM ticket-flow engine that stores pending ticket state inside `Conversation.metadata.crmTicketFlow`.
- Technical support, sales, booking, complaints, and human-agent requests now start a CRM flow instead of creating an official ticket immediately.
- Official tickets are created only after collecting all required fields:
  - customer name
  - phone number
  - issue/request description
- Customer-facing replies for missing fields and ticket confirmation are generated by the AI using the customer language, bot tone, emoji style, business category prompt, and tenant custom instructions.
- The code no longer appends fixed Arabic/English ticket confirmation text to the customer reply.

## Context-switch handling

If a customer starts a ticket flow, then asks a knowledge/product/service question before completing the fields, the flow is paused for context and the assistant answers the current question first. The pending ticket flow remains available for the next customer message.

Example:

1. Customer: I want to buy.
2. Bot collects required CRM fields.
3. Customer: Show me products first.
4. Bot answers products from knowledge and does not force ticket fields in that reply.
5. Customer can later provide name/phone/request description and the ticket is created once.

## Duplicate prevention

- Repeated support/sales/booking phrases reuse the same pending flow in the same conversation.
- Existing open/pending/in-progress ticket dedupe remains in `ensureTicketForConversation`.
- Lead sync continues to dedupe by normalized phone number and now also reads `metadata.customerPhone` from tickets.

## Bot settings foundation

Added additional AI setting fields:

- languageMode
- tonePreset
- warmthLevel
- salesStyle
- supportStyle
- emojiStyle
- businessCategory
- businessSubcategory
- categoryPromptEn
- customInstructionsEn
- ticketPolicy
- leadPolicy

The AI settings page now exposes additional controls for tone, emoji style, sales/support style, category/subcategory, and English custom instructions.

## Business categories

Added bilingual category catalog:

- Healthcare
- Retail
- Technology
- Professional services
- Education
- Real estate
- Automotive
- Restaurants and food
- Home services
- Travel and hospitality

Available through:

`GET /api/business-categories`

Registration accepts:

- businessCategory
- businessSubcategory

and creates default AI settings with a category-specific English prompt.

## Billing usage and grace period

- Tenant subscriptions now include `graceMessageLimit`, default 20 messages.
- AI sending is allowed until plan allowance + grace messages.
- Realtime usage event is published when usage reaches 80% or higher.
- Dashboard realtime bridge displays a red billing toast when usage is near/over the plan message limit.

## Important implementation principle

Customer-visible CRM/ticket messages are not hardcoded. The code only stores internal state, required field names, categories, and events. The AI generates customer-facing language according to settings and context.
