Task 4.1 — Complete Instagram Adapter

الهدف

Instagram Messaging يجب أن يعمل عبر Meta Graph API بنفس abstraction الخاص بالقنوات.

المطلوب

أنشئ أو أكمل:

src/server/channels/providers/instagram.ts

يجب أن يدعم

Webhook Verification

* استخدم نفس HMAC-SHA256 pattern الخاص بـ Meta:

X-Hub-Signature-256

* استخدم per-channel appSecret أو fallback آمن من env.
* لا تستخدم return true.

Normalize Incoming

ادعم Instagram DM payloads من Meta Webhooks:

entry[].messaging[]

واستخرج:

externalEventId
externalMessageId
externalUserId
text
attachments
timestamp
customer profile if available

Send Message

استخدم Meta Graph API:

POST https://graph.facebook.com/v18.0/me/messages
Authorization: Bearer {PAGE_ACCESS_TOKEN}

Payload MVP:

{
  "recipient": { "id": "IG_SCOPED_USER_ID" },
  "message": { "text": "message text" }
}

Error Handling

تعامل مع:

* invalid token
* permissions missing
* recipient not reachable
* outside messaging window
* rate limit
* unsupported attachment
* app review required

Channel Config

Instagram channel يجب أن يستخدم:

instagramBusinessId
linkedPageId
username
pageAccessTokenEncrypted
appSecretEncrypted or appSecret reference
verifyToken
permissions

ممنوع

* تخزين token plaintext.
* طباعة token.
* fake success.

Task 4.3 — Fix AI Quota Race Condition

المشكلة

Day 1 اكتشف أن:

assertCanSendAiMessage()
ثم
recordAiMessageUsage()

يمكن أن يحدث بينهما race condition مع أكثر من worker.

المطلوب

أنشئ أو حسّن:

src/lib/quota.ts

باستخدام Redis atomic operations.

السلوك المطلوب

* مفتاح شهري:

quota:ai_messages:{tenantId}:{YYYY-MM}

* قبل إرسال AI reply:

INCR
إذا تجاوز limit → rollback/decrement أو reject safely

* استخدم Lua script أو multi/transaction إذا لزم.
* TTL حتى نهاية billing period أو نهاية الشهر.
* sync دوري إلى MongoDB TenantSubscription.usedMessages.

قواعد

* لا تجعل counters تعتمد على read ثم write.
* لا تسمح بتجاوز limit تحت concurrency.
* إذا Redis down:
    * fail closed للخطة المجانية/المحدودة.
    * أو fallback آمن مع logging واضح حسب قرارك.
* وثّق القرار.

Task 4.4 — Make AI Context Token-Aware

المشكلة

AI context يعتمد على آخر 10 رسائل فقط بدون token budget.

المطلوب

بدل:

last 10 messages

استخدم:

recent messages until token budget reached

المتطلبات

* استخدم estimateTokens() إن موجود.
* احسب budget لـ:
    * system prompt
    * agent role prompt
    * knowledge context
    * conversation context
    * tool instructions
* لا تتجاوز model context limit.
* default budget آمن مثل:
    * 4000 tokens للموديلات الصغيرة.
    * configurable لكل model/provider إن متاح.
* لو المحادثة طويلة:
    * استخدم summary إن موجود.
    * أو أضف placeholder واضح لـ future summarization.

مخرجات

* جودة الرد أفضل.
* أخطاء context length تقل.
* لا تكسر ai-agent.service.

Task 4.5 — SSE Realtime Scaling via Redis Pub/Sub

المشكلة

Realtime inbox يستخدم in-memory EventEmitter، وهذا لا يعمل مع multi-pod أو horizontal scaling.

المطلوب

أنشئ:

src/lib/realtime.ts

أو حسّن الموجود.

السلوك المطلوب

* Publish events إلى Redis channel:

inbox:{tenantId}:events

* SSE endpoint يشترك في نفس channel.
* عند:
    * new message
    * conversation updated
    * assignment changed
    * delivery status changed
        يتم publish للـ tenant channel.

قواعد

* tenant isolation في channel name.
* لا ترسل أحداث tenant إلى tenant آخر.
* حافظ على fallback dev لو Redis غير متاح، لكن واضح ومحدود.
* لا تكسر SSE API الحالي.
