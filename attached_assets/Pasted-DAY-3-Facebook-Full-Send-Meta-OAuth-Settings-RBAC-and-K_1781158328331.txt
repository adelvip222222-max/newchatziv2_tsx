Task 3.1 — Implement Facebook Messenger sendMessage

الهدف

Facebook Messenger outbound يجب أن يكون حقيقيًا وليس stub.

المطلوب

نفذ sendMessage() في:

src/server/channels/providers/facebook.ts

باستخدام Meta Graph API:

POST https://graph.facebook.com/v18.0/me/messages
Authorization: Bearer {PAGE_ACCESS_TOKEN}

Payload MVP:

{
  "recipient": { "id": "PSID" },
  "message": { "text": "message text" },
  "messaging_type": "RESPONSE"
}

يجب دعم

* text messages.
* parsing message_id من Meta response.
* clear error handling.
* rate limit handling.
* invalid token handling.
* 24h messaging window error handling.
* لا تطبع page access token في logs.

قواعد

* token يجب أن يكون encrypted at rest ويفك server-side فقط.
* لو pageAccessToken غير موجود: return failure واضح.
* لو pageId غير موجود: return failure واضح.
* لا تستخدم fake success.

Task 3.2 — Complete Meta OAuth Flow

الهدف

ربط Facebook/Instagram بطريقة OAuth آمنة، server-side، بدون إرسال tokens للفرونت.

راجع الموجود

افحص:

src/app/api/oauth/meta/route.ts
src/app/dashboard/channels/
src/lib/models/channel.ts
src/server/channels/
src/lib/crypto.ts

OAuth Flow المطلوب

User clicks Connect Meta
→ server generates signed short-lived state
→ state includes tenantId + userId + nonce + returnUrl
→ redirect to Meta OAuth dialog
→ callback receives code + state
→ validate state server-side
→ exchange code for access token server-side
→ fetch /me/accounts
→ fetch linked Instagram accounts for pages
→ show selectable page/account list OR create pending connections
→ user selects page/IG account
→ encrypt tokens
→ create Channel records
→ subscribe page to webhooks
→ activate channel

Security Rules

* لا ترجع access token في postMessage.
* لا تخزن token في localStorage/sessionStorage.
* لا ترسله للفرونت.
* token encryption إلزامي.
* state must expire.
* state must be single-use if practical.
* verify tenant ownership.
* verify user has permission settings.manage or channels.manage.

Channel Records

Facebook channel config يجب أن يشمل:

pageId
pageName
pageAccessTokenEncrypted
appSecretEncrypted or appSecret reference
verifyToken
webhookSubscribedAt
permissions

Instagram channel config يجب أن يشمل:

instagramBusinessId
username
linkedPageId
pageAccessTokenEncrypted
verifyToken
webhookSubscribedAt
permissions

Webhook Subscription

بعد إنشاء channel حاول subscribe للصفحة:

POST /{page-id}/subscribed_apps

مع fields المناسبة:

messages
messaging_postbacks
message_deliveries
message_reads

إذا فشل بسبب permissions/app review:

* لا تفشل النظام كله.
* اجعل channel status = needs_permissions أو webhook_subscription_failed.
* اعرض رسالة واضحة في UI.
* وثّق المطلوب في docs.

Task 3.5 — Move Knowledge Training to BullMQ

المشكلة

Knowledge training يعمل synchronously داخل request ويسبب timeout.

المطلوب

أنشئ queue:

knowledge-training-queue

ملفات متوقعة:

src/lib/queues/index.ts
workers/knowledge-worker.ts
src/lib/knowledge.ts
src/app/api/knowledge/

السلوك الجديد

عند رفع document:

create KnowledgeDocument(status="pending")
enqueue training job
return immediately to UI
worker processes:
  extract text
  clean text
  chunk
  embed
  insert chunks
  update document status="ready"

retrainAllKnowledge

بدل التشغيل المتسلسل:

enqueue one job per document
batch jobs
status يظهر processing

Error Handling

* failed job → document.status = “error”
* store error message safely.
* retries with backoff.
* لا تسقط document بصمت.

Task 3.7 — Fix Embedding Dimension Mismatch

المشكلة

OpenAI embeddings = 1536 dimensions
local hash = 128 dimensions
الكود يقارن بينهم باستخدام Math.min() وهذا يعطي scores خاطئة.

المطلوب

* أثناء search، استخدم فقط chunks بنفس embeddingProvider ونفس dimensions.
* إذا query embedding من OpenAI:
    * ابحث فقط في chunks embeddingProvider = openai
* إذا query embedding local:
    * ابحث فقط في chunks embeddingProvider = local-hash
* لو OpenAI key أضيف لاحقًا:
    * علّم local-hash chunks كـ needsRetraining=true أو أضف report واضح.

ممنوع

* المقارنة بين vectors بأبعاد مختلفة.
* تجاهل mismatch بصمت.
